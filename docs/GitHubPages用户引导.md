## 两种模式怎么选
- 本地模式：机体托管 llama.cpp server，调用同级目录 `EVA_BACKEND/<arch>/<os>/<device>/<proj>` 下的可执行文件，模型来自 `EVA_MODELS/llm/*.gguf`。优点：离线、低时延；要求有可用后端与模型文件。
- 链接模式：直连 OpenAI 兼容端点（/v1/chat/completions 等），填写 endpoint/key/model 即可。优点：零本地负载；依赖网络与远端算力。
- 提示：首次启动如未找到配置，会自动扫描 EVA_MODELS 选择最小 gguf 模型与嵌入模型并写入配置；Qt 正则禁用 static QRegularExpression 以避免旧 CPU 的 PCRE2 JIT SIGILL。

## 本地模式装载步骤
1. 点击“装载”→选“本地模式”。
2. 选择 gguf 模型（默认按最小模型预填），确认后进入“装载中”（界面锁定+转轮）。
3. 后台按当前设备/端口设置启动或重启 llama.cpp server，自动切换到本地端点并新建会话。
4. 就绪后界面变为“正常”，可直接聊天或进入下一步（语音/视觉/工具）。
5. 首次装载会根据显存推算 GPU offload，装载后显示实际 n_layer+1 修正值。

## 链接模式配置步骤
1. 点击“装载”→选“链接模式”。
2. 填写远端 endpoint/key/model，确认后切换为远端模式并停止本地服务。
3. 系统自动清屏、注入系统指令并解锁，可直接聊天。
4. 如需工具/增殖，同样在“约定”启用后继续使用。

## 快速开始（3 步）
1. 先完成“本地模式”或“链接模式”装载。
2. 打开 [对话：开局即用](pages/01-对话.md)，发送首条消息（可带图/音频）。
3. 按需继续 12 步页面，逐项体验（补完、语音、视觉、文生图、知识库、MCP、系统工程师、沙盒、技能、架构师、机体控制）。
