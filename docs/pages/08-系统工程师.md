# 08 系统工程师：命令与编程协作

[返回总览](GitHubPages用户引导.md) · [上一页：MCP](07-MCP.md) · [下一页：使用沙盒](09-使用沙盒.md)

## 🎯 适用场景
- 让模型代执行命令、读写文件、跑测试或编写脚本。
- 本地项目协作，减少手动敲命令。

## 📦 你会得到
- 启用系统工程师工具的流程与常用命令型工具说明。
- 如何安全限制工作目录与命令范围。

## 🛠️ 前置准备
1. 在“约定”勾选“系统工程师”，设定“工程工作目录”（默认 `EVA_WORK`）。
2. 如有 AGENTS.md/SKILL.md，可放入工作目录以便模型遵循。

## 🚀 操作步骤
1. 保存约定，自动重置上下文并加载工具。
2. 在对话提出任务，例如“修复 src/main.cpp 编译错误并给出补丁”。
3. 模型可能发起：
   - `execute_command`：执行命令，返回 stdout/stderr。
   - `read_file/write_file/replace_in_file/edit_in_file/list_files/search_content`：文件读写与查找。
   - `ptc`：一次性提交脚本，写入 ptc_temp 后执行。
4. 结果以 `tool_response` 回填，模型继续直到任务完成。

## 🧠 小技巧
- 复杂流程用 `ptc` 一次性交付脚本；注明运行目录、依赖、预期输出。
- 先看再改：要求模型先 `read_file/list_files`，确认后再写。
- 高风险路径先复制到工作目录再修改，降低误改风险。

## ❓ 常见问题
- 权限/路径错误：检查工作目录；在提问中给出相对路径。
- 输出过长：要求“只返回要点和必要日志”，长日志写入文件。
- 命令找不到：提示模型安装或检查依赖（环境允许的前提下）。
