# 06 知识库：你的私有仓库

[返回总览](GitHubPages用户引导.md) · [上一页：文生图](05-文生图.md) · [下一页：MCP](07-MCP.md)

## 🎯 适用场景
- 让模型基于私有资料回答，减少遗漏与幻觉。
- FAQ、产品手册、代码注释、长报告的问答。

## 📦 你会得到
- “启动嵌入服务→入库→检索”的完整流程。
- 在“约定”启用知识库工具的要点。

## 🛠️ 前置准备
1. 在“增殖-知识库”选择嵌入模型并启动（`--embedding` 模式）。
2. 准备要入库的文本，按段/章节拆分更易检索。

## 🚀 操作步骤
1. 在“增殖-知识库”上传或粘贴文本，按段入库（调用 `/v1/embeddings` 写入向量库）。
2. 在“约定”勾选“知识库”工具，保存后重置上下文生效。
3. 回到对话提出问题，模型会自动检索并返回匹配片段。
4. 继续对话时，知识库保持可用，直到关闭或重置。

## 🧠 小技巧
- 清晰命名和分段，检索更准；需要引用可要求“标注来源/匹配度”。
- 回答太啰嗦：让模型“只基于引用回答，不编造”。
- 需要多文档对比：在提问中指明文档名或关键段落。

## ❓ 常见问题
- 无匹配：检查嵌入服务是否启动、文档是否入库；可调大 Top-N。
- 维度不符：查看嵌入服务日志确认模型维度，必要时重启清库。
- 结果偏离事实：让模型先列引用片段，再据此回答。
