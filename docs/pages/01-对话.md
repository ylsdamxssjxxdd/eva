# 01 对话：开局即用

[返回总览](../GitHubPages用户引导.md) · [下一页：补完](02-补完.md)

## 适用场景
- 第一次上手 EVA，想直接和模型聊天、获取答案。
- 日常问答、代码解惑、方案讨论，或让模型基于图片/音频给出反馈。

## 你会得到
- 完整的“装载→发送→流式回复”体验。
- 知道五个按钮、状态提示和快捷键怎么用。

## 前置准备（1 分钟）
1. 确认本地已有可用模型（本地模式），或已有远端 API endpoint/key/model（链接模式）。
2. 若走本地模式，确保同级目录存在 `EVA_BACKEND/**` 与 gguf 模型（可放在 `EVA_MODELS/llm`）。

## 操作步骤
1. 打开 EVA，点击“装载”。
2. 选择“本地模式”或“链接模式”：
   - 本地模式：选择 gguf 模型，确认后进入“装载中”，等待转轮结束即可。
   - 链接模式：填入 endpoint/key/model，点击确认。
3. 待界面变为“正常”状态后，在输入框键入消息。
4. 按 `Ctrl+Enter` 或点击“发送”。若要带图，直接拖拽图片/按 F1 截屏后发送；带音频则上传 WAV/MP3/OGG/FLAC。
5. 观察底部速度与上下文占用提示，回复结束后继续下一轮对话即可。

## 界面提示与捷径
- 五按钮：装载 / 约定 / 设置 / 重置 / 发送。推理中仅“重置”可用。
- 快捷键：F1 截屏、F2 录音、Ctrl+Enter 发送。
- 状态灯：装载前（仅装载可点）、装载中（全禁用）、正常（全可用）、推理中（仅重置）、录音中（F2 控制）。

## 常见问题
- 装载卡住：检查模型路径与权限；本地模式看“模型日志”是否出现 listening/on/at。
- 无回复：确认 endpoint 填写到 `/v1` 根，不要被代理拦截 SSE；模型名存在且密钥有效。
- 图片/音频过大：建议压缩后再发；多图可分条发送。

## 验收要点
- 能在“正常”状态下连续多轮对话。
- 流式回复速度和上下文占用显示正常。
- 截图/音频能随消息一起发送。
