# 04 视觉交互：看图说话

[返回总览](GitHubPages用户引导.md) · [上一页：语音交互](03-语音交互.md) · [下一页：文生图](05-文生图.md)

## 🎯 适用场景
- 让模型解读截图、表格、界面或照片。
- 定位报错/界面问题，直接传截图让模型分析。

## 📦 你会得到
- 从上传/截屏到问答的完整流程。
- 本地视觉模型（mmproj）挂载要点。

## 🛠️ 前置准备
1. 本地模式：在“设置”选择视觉编码模型 mmproj（若同目录有匹配 GGUF 会自动挂载）。
2. 准备要分析的图片，或使用 **F1** 截屏。

## 🚀 操作步骤
1. 在输入框拖拽图片或右键上传；或按 **F1** 截屏自动附加。
2. 在文本框描述需求，如“提取表格中的关键数字”或“解释这条报错”。
3. 点击 **“发送”**，等待视觉预解码 + 文本回答。
4. 多张图建议分条发送，逐步提问。

## 🧠 小技巧
- 表格/代码图：要求“逐行说明”或“提取关键信息”。
- UI/报错：补充当前操作、预期结果，方便模型给方案。
- 处理超长图：裁剪重点区域或多段发送，提升准确度。

## ❓ 常见问题
- 模型说看不到图：确认消息已附图；检查日志中是否有 `image_url`。
- 结果模糊：图太压缩或过长，尝试裁剪清晰区域。
- 本地模式慢：检查 mmproj 是否正确、显存是否足够，必要时降分辨率。
