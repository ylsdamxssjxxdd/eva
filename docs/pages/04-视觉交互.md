# 04 视觉交互：看图说话

[返回总览](../GitHubPages用户引导.md) · [上一页：语音交互](03-语音交互.md) · [下一页：文生图](05-文生图.md)

## 适用场景
- 让模型解读截图、表格、界面或照片。
- 配合截图快速定位问题、描述 UI、识别报错。

## 你会得到
- 从上传/截屏到提问的完整流程。
- 本地视觉模型（mmproj）挂载提示。

## 前置准备
1. 本地模式：在“设置”中选择 mmproj（视觉编码）gguf 模型；若同目录下有匹配的 mmproj 文件，EVA 会尝试自动挂载。
2. 准备要分析的图片，或使用 F1 直接截屏。

## 操作步骤
1. 在输入框拖拽图片，或右键上传；也可按 F1 截图后自动附加。
2. 在文本框描述你想让模型做什么，例如“请总结这张报表的关键数字”。
3. 点击“发送”，等待视觉预解码 + 文本问答完成。
4. 如果多张图，建议分多条消息发送，逐步提问。

## 进阶提示
- 表格/代码截图：在提问中强调“逐行说明”或“提取关键数字”。
- UI/报错截图：说明当前操作与期望结果，模型能结合画面做故障判断。
- 本地模式慢：检查 mmproj 是否正确、显存是否足够，可降低分辨率再发。

## 常见问题
- 模型说看不到图片：确认发送时选中了图片，或检查日志中是否有 image_url 字段。
- 结果模糊：图片太压缩/过长，尝试裁剪重点区域。
- 视觉模型未挂载：在设置里重新选择 mmproj，必要时重启装载。

## 验收要点
- 图片能随消息发送，模型输出能引用图中信息。
- F1 截屏可以直接进入消息并被识别。
