# 09 使用沙盒：把高风险关进盒子

[返回总览](GitHubPages用户引导.md) · [上一页：系统工程师](08-系统工程师.md) · [下一页：使用技能](10-使用技能.md)

## 🎯 适用场景
- 在隔离环境中运行高风险命令、拉取依赖或重编译，保护主机。
- 预验证脚本/技能包的安全性，再迁回主环境。

## 📦 你会得到
- 启动容器沙盒并绑定工程师工具的流程。
- 挂载目录、端口映射与镜像选择的提示。

## 🛠️ 前置准备
1. 主机安装 Docker（或使用内置镜像模式）。
2. 准备常用基础镜像，或确保本地镜像列表可用。

## 🚀 操作步骤
1. 打开“增殖”或“设置”的沙盒页签。
2. 选择本地镜像或输入基础镜像名（如 `ubuntu:latest`），必要时先拉取。
3. 配置挂载目录（将工作目录映射到容器）、端口映射（如需要网络访问）。
4. 启动沙盒，在系统工程师工具中选择“使用沙盒”或绑定该容器。
5. 随后命令将在容器内执行，输出同样回填到对话。
6. 任务结束后，可一键重置/删除容器，环境恢复干净。

## 🧠 小技巧
- 预热：提前拉取镜像，减少首次等待；可制作带依赖的自定义镜像。
- 只读挂载：对敏感目录使用只读挂载，避免意外写入。
- 先测后迁：在沙盒跑通流程后，再让模型在主机目录重放一次（可选）。

## ❓ 常见问题
- 拉取慢：检查网络或换用近源镜像仓库。
- 容器缺依赖：让模型在容器内安装，或改用预装依赖的镜像。
- 看不到输出：确认工程师工具已绑定到沙盒实例。
