# 09 使用沙盒：把高风险关进盒子

## 你能完成
- 在 Docker 沙盒中运行命令/脚本，隔离高风险操作与依赖安装。
- 绑定系统工程师工具，让模型的读写/命令全部落在容器内。
- 自动修复缺失挂载，确保 `/eva_workspace`、`/eva_skills` 可用。

## 开始前
- 主机已安装 Docker，能拉取/运行镜像。
- 打开“约定”或“设置”沙盒区域，选择模式：`None`（关闭）、`Image`（基于镜像新容器）、`Container`（复用已存在容器）。
- 准备基础镜像（如 `ubuntu:latest`），或已有挂载好的容器。

## 操作步骤
1. 在“约定/设置”选择沙盒模式与镜像/容器，保存后会重置并锁定 UI，等待沙盒就绪。
2. 如果选择 Image，EVA 会拉取镜像并创建容器，自动挂载当前工程师工作目录到 `/eva_workspace`，技能目录到 `/eva_skills`。
3. 看到“ready”后解锁：此时系统工程师工具的命令将运行在容器内，路径以 `/eva_workspace` 为根。
4. 若提示缺少挂载，按提示修复或重新选择容器；需要退出可将模式切回 `None`。

## 常见问题
- **拉取慢/失败**：检查网络与镜像源；必要时手工 docker pull 后再选择容器模式。
- **缺挂载**：容器模式下若未挂载 `/eva_workspace`，EVA 会弹窗提示并自动重建（需确认）。
- **命令仍在宿主执行**：检查约定是否保存且工程师工具已勾选，状态区应显示容器名称与挂载路径。
