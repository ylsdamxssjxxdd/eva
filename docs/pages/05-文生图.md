# 05 文生图：把想法拉成画面

[返回总览](../GitHubPages用户引导.md) · [上一页：视觉交互](04-视觉交互.md) · [下一页：知识库](06-知识库.md)

## 适用场景
- 需要插画、封面、角色设定、概念草图。
- 想把文字 prompt 直接变成图片。

## 你会得到
- 在“增殖-文生图”完成一次生成的流程。
- Prompt 书写与风格模板的小技巧。

## 前置准备
1. 在“增殖-文生图”面板确认扩散模型路径（默认 `text2image/sd1.5-anything-3-q8_0.gguf`）。
2. 若需特定分辨率或风格，准备好对应模板（可在面板中选择）。

## 操作步骤
1. 打开“增殖-文生图”。
2. 输入你的提示词（如“暖色调、科幻驾驶舱、全身角色”），可附加负面提示词。
3. 选择分辨率、步数、CFG、风格模板。
4. 点击“生成”，等待输出。生成后的图片会在输出区以标记+路径展示。
5. 若想继续调整，修改提示词或分辨率再次生成。

## 进阶提示
- 先在对话区让模型帮写精炼的 prompt，再粘贴到文生图面板。
- 分辨率越高越耗时，显存不足时可先用 512 边长测试。
- 想要更写实或特定风格，可在提示词中明确镜头、光线、风格名。

## 常见问题
- 生成失败：查看输出区的错误片段，检查模型路径和显存；必要时降低分辨率或步数。
- 图片路径太长：输出区已自动换行，直接复制标注路径查看即可。
- 想保存：直接在路径位置打开文件，或在对话中让模型帮改名整理。

## 验收要点
- 能成功生成至少一张图片，路径正确。
- 调整提示词/分辨率后可再次生成。
