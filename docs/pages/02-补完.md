# 02 补完：初始形态

[返回总览](GitHubPages用户引导.md) · [上一页：对话](01-对话.md) · [下一页：语音交互](03-语音交互.md)

## 🎯 适用场景
- 长文续写、邮件/报告/方案一键补完。
- 代码片段补全、注释补齐、脚本快速成稿。
- 希望“说一句，出一段”，避免多轮反复。

## 📦 你会得到
- 一次性输出的完整段落/代码，不受上一轮对话影响。
- 更稳的采样策略（低温度）和长度控制技巧。

## 🛠️ 前置准备
1. 已完成“对话”页的装载（本地或链接均可）。
2. 在“设置”中将温度调低至 0.3~0.6，避免发散；按需设置 `n_predict` 控制补完长度。

## 🚀 操作步骤
1. 在主界面右上角切换到 **“补完”** 状态（与“对话”区分）。
2. 将待续写/补全的文本粘贴到输入框，必要时附加图片/音频作为上下文。
3. 点击 **“发送”** 或 `Ctrl+Enter`，模型会一次性给出补全文本。
4. 如需再次补完，修改输入后重复发送即可。

## 🧠 小技巧
- 代码补全：在开头注明语言、文件名或函数名，模型更易对齐风格。
- 防啰嗦：在文本末尾写明“到此结束”或限制字数，如“请 200 字内收尾”。
- 防跑题：在输入首行补一句“上下文/目标”，让模型聚焦。

## ❓ 常见问题
- 输出过短：提高 `n_predict`，或在提示里要求“生成完整段落”。
- 输出跑题：先用“对话”模式让模型总结要点，再把总结贴回补完。
- 想多轮：补完模式不记忆上一轮，需切回“对话”以保持上下文。
