# 02 补完：精准一击

[返回总览](../GitHubPages用户引导.md) · [上一页：对话](01-对话.md) · [下一页：语音交互](03-语音交互.md)

## 适用场景
- 长文续写、代码片段补全、邮件模板快速生成。
- 需要一次性输出而非多轮对话。

## 你会得到
- 清晰的“补完”模式开关与操作路径。
- 让输出更稳定的采样参数建议。

## 前置准备
1. 已完成“对话”页面中的装载流程（本地或远端均可）。
2. 建议在“设置”中调低温度（如 0.3~0.6）以提高确定性。

## 操作步骤
1. 在主界面右上角切换到“补完”状态（与“对话”区分）。
2. 将要续写/补全的文本粘贴到输入框，可同时附加图片/音频辅助理解。
3. 点击“发送”或按 `Ctrl+Enter`。此模式下模型会一次性输出，无需上下文多轮。
4. 如需再次补完，替换输入内容后重复步骤。

## 进阶提示
- 温度越低越可控，top_k/top_p 可按默认或略降；n_predict 控制补完长度。
- 如果模型倾向于重复，可在输入末尾添加明确的收尾提示，例如“请在此结束。”。
- 需要代码片段时，可在开头说明语言与期望的函数/文件名。

## 常见问题
- 输出过短：适当提高 n_predict 或在提示中要求“完整输出到结束”。
- 输出跑题：在输入前加一行上下文说明；或先用“对话”模式让模型总结要点，再切回补完。
- 多轮想继续：补完模式并不记忆上一轮，建议切回“对话”模式继续交流。

## 验收要点
- 可以一键获得完整段落或代码，长度可控。
- 切换“补完/对话”后状态栏提示同步更新。
